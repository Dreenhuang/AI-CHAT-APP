## 🚀 快速开始

### 方式一：一键启动（推荐）

双击运行 **`管理后台系统_一键启动器.bat`**

脚本会自动：
- ✅ 检查Bun环境
- ✅ 启动后端API服务（端口9450）
- ✅ 启动前端管理界面（端口5173）
- ✅ 自动打开浏览器访问 http://localhost:5173

### 方式二：手动启动

#### 1️⃣ 启动后端服务

```bash
cd admin-backend
bun run src/index.js
```

**验证后端启动成功**：
```
✅ Server listening at http://localhost:9450
✅ API文档地址: http://localhost:9450/docs
```

#### 2️⃣ 启动前端服务

```bash
cd admin-frontend
bun run dev
```

**验证前端启动成功**：
```
✅ VITE v5.4.21 ready in XXX ms
✅ Local:   http://localhost:5173/
```

- 渐变色装饰圆圈
- 流畅的加载动画

**对接API**:
```javascript
POST /api/admin/v1/auth/login
{
  "username": "admin",
  "password": "Admin@123456"
}
// 返回: { token, refreshToken, user }
```

### 3️⃣ 📝 议题管理系统

**页面路径**: `/topics`  
**核心功能**:

#### 列表展示
- 表格形式：ID、标题、状态、类型、时间、操作
- 多选批量操作
- 分页控制（10/20/50条每页）
- 排序功能

#### 搜索筛选
- 关键词搜索（300ms防抖）
- 状态筛选：进行中/已完成/已暂停/待审核
- 类型筛选：辩论/投票/讨论/问答
- 已选条件显示为Tag（可移除）

#### CRUD操作
- **新建**: 模态框表单，支持图片上传、标签多选
- **编辑**: 复用模态框，自动填充现有数据
- **查看**: 右侧滑出抽屉，显示完整信息+统计
- **删除**: 二次确认对话框

**状态标签颜色**:
| 状态 | 颜色 | 色值 |
|------|------|------|
| 进行中 | 蓝色 | `#0071e3` |
| 已完成 | 绿色 | `#34c759` |
| 已暂停 | 橙色 | `#ff9f0a` |
| 待审核 | 灰色 | `#86868b` |

**对接API**:
```javascript
GET    /api/admin/v1/topics           // 列表
POST   /api/admin/v1/topics           // 创建
PUT    /api/admin/v1/topics/:id       // 编辑
DELETE /api/admin/v1/topics/:id       // 删除
PATCH  /api/admin/v1/topics/:id/status // 上架/下架
```

## 🔌 API接口文档

### 基础信息

- **Base URL**: `http://localhost:9450`
- **API Prefix**: `/api/admin/v1`
- **认证方式**: `Authorization: Bearer <token>`
- **响应格式**: `{ code: 200, message: "success", data: {...} }`

### 核心接口清单

#### 认证模块 (`/auth`)
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/login` | 管理员登录 |
| GET | `/auth/profile` | 获取当前用户信息 |
| PUT | `/auth/password` | 修改密码 |
| POST | `/auth/logout` | 用户登出 |
| POST | `/auth/refresh` | 刷新Token |

#### 仪表盘 (`/dashboard`)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/dashboard/metrics` | 核心统计指标 |
| GET | `/dashboard/user-trend` | 用户增长趋势 |
| GET | `/dashboard/topic-distribution` | 议题分布 |
| GET | `/dashboard/health` | 系统健康检查 |

#### 议题管理 (`/topics`)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/topics` | 议题列表（分页/筛选） |
| GET | `/topics/:id` | 议题详情 |
| POST | `/topics` | 创建议题 |
| PUT | `/topics/:id` | 编辑议题 |
| DELETE | `/topics/:id` | 删除议题 |
| PATCH | `/topics/:id/status` | 上架/下架 |
| PATCH | `/topics/:id/hotness` | 调整热度 |

#### 用户管理 (`/users`)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/users` | 用户列表 |
| GET | `/users/search` | 用户搜索 |
| GET | `/users/:id` | 用户详情 |
| PATCH | `/users/:id/status` | 修改状态 |

#### 审计日志 (`/audit-logs`)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/audit-logs` | 日志列表 |

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/configs` | 配置项列表 |
| GET | `/configs/history` | 变更历史 |

| PUT | `/configs/batch` | 批量更新 |
| POST | `/configs/:key/reset` | 重置为默认 |

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/souls` | 角色列表 |
| GET | `/souls/:id` | 角色详情 |

### 环境要求

- **Node.js**: >= 18.0.0
- **Bun**: >= 1.0.0（推荐包管理器和运行时）
- **Git**: 版本控制
- **浏览器**: Chrome/Firefox/Safari 最新版

### 安装依赖

```bash
# 后端依赖
cd admin-backend
bun install

# 前端依赖
cd ../admin-frontend
bun install
```

### 开发模式

```bash
# 终端1：启动后端（端口9450）
cd admin-backend
bun run src/index.js --watch

# 终端2：启动前端（端口5173）
cd admin-frontend
bun run dev
```

### 生产构建

```bash
# 构建前端
cd admin-frontend
bun run build

# 输出到 dist/ 目录
```

### 代码规范

- **组件命名**: PascalCase（如 `UserTable.tsx`）
- **文件组织**: 每个功能模块独立文件夹
- **样式方案**: Tailwind CSS原子类 + Design Token变量
- **状态管理**: React Hooks (useState/useEffect)
- **API调用**: 统一通过 `src/services/api.js`
- **错误处理**: try/catch + 用户友好提示

## ❓ 常见问题 FAQ

### Q1: 登录失败提示"网络连接失败"

**原因**: 后端服务未启动或端口不匹配

**解决方案**:
1. 确认后端服务正在运行：访问 http://localhost:9450/docs
2. 检查 `.env` 文件中的 `VITE_API_BASE_URL` 是否正确
3. 尝试重启后端服务

---

### Q2: 页面显示空白或报错

**可能原因**:
- 前端编译错误
- 缺少环境变量
- 路由配置错误

**解决方案**:
1. 查看浏览器控制台(F12)的错误信息
2. 确认所有依赖已安装：`bun install`
3. 重启开发服务器

---

### Q3: 如何修改默认端口？

**前端端口**（当前5173）:
编辑 `admin-frontend/package.json`:
```json
{
  "scripts": {
    "dev": "vite --port 你的端口 --host"
  }
}
```

**后端端口**（当前9450）:
编辑 `admin-backend/.env`:
```
PORT=你的端口
```

记得同步更新 `admin-frontend/.env` 的 `VITE_API_BASE_URL`

---

### Q4: 如何添加新的功能模块？

步骤：
1. 在 `src/pages/` 创建新文件夹
2. 编写页面组件（参考现有页面结构）
3. 在 `App.jsx` 添加路由配置
4. 在 `Sidebar.jsx` 添加导航菜单项
5. （可选）在后端添加对应的API路由

---

### Q5: 如何切换到生产数据库？

编辑 `admin-backend/.env`:
```
DATABASE_URL=postgresql://user:password@host:5432/database_name
```

然后执行数据库迁移：
```bash
cd admin-backend
bunx prisma migrate deploy
```
