## 二、后端部署方案

### 2.1 移动端独立后端

根据需求，移动端需要**独立部署后端**。

| 项目 | 内容 |
|------|------|
| **后端类型** | Node.js + Express |
| **端口** | **9461**（已锁定） |
| **WebSocket** | ws |
| **部署位置** | 腾讯云（与Web版独立部署） |

### 2.2 端口分配

**端口池：**
- 单服务：9450, 9461, 9473, 9516, 9528, 9632, 9741, 9852
- 多服务连续端口段：9450-9453, 9460-9463, 9510-9513, 9620-9623

**检测结果：**
| 端口 | 状态 |
|------|------|
| 9450 | 被占用 |
| **9461** | **可用（已锁定）** |
| 9473 | 可用（备用） |
| 9516 | 可用（备用） |

**端口管理策略：**
- 首选端口：9461
- 冲突处理：如果端口被占用，自动切换到下一个可用端口
- 杀进程重启：如果需要，先kill原进程再重启

### 2.3 后端API设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查 |
| `/api/auth/send-code` | POST | 发送验证码 |
| `/api/auth/login` | POST | 手机号+验证码登录 |
| `/api/auth/register` | POST | 注册 |
| `/api/topics` | GET | 获取预设议题（200个） |
| `/api/topics/random` | GET | 获取随机20个 |
| `/api/groups` | GET | 获取群组列表 |
| `/api/groups` | POST | 创建群组 |
| `/api/groups/:id` | GET | 获取群组详情 |
| `/api/groups/:id/members` | GET/POST/DELETE | 群组成员管理 |
| `/api/souls` | GET | 获取Soul好友 |
| `/api/souls` | POST | 添加Soul好友 |
| `/api/souls/:id` | DELETE | 删除Soul好友 |
| `/api/debates` | GET | 获取辩论历史 |
| `/api/debates` | POST | 创建新辩论 |
| `/api/debates/:id` | GET | 获取辩论详情 |
| `/ws` | WebSocket | 实时讨论 |

### 2.4 WebSocket消息协议

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| `debate:start` | 客户端->服务端 | 开始辩论 |
| `debate:message` | 双向 | 辩论消息 |
| `debate:stream:chunk` | 服务端->客户端 | 流式输出 |
| `debate:complete` | 服务端->客户端 | 辩论完成 |
| `debate:stop` | 客户端->服务端 | 停止辩论 |

## 四、数据同步策略

### 4.1 本地优先 + 云端备份

| 场景 | 策略 |
|------|------|
| **写入** | 先写入本地AsyncStorage，同时异步同步到云端 |
| **读取** | 先读本地，本地无数据时从云端拉取 |
| **冲突** | 以云端数据为准（多设备场景） |
| **离线** | 支持本地写入，联网后自动同步 |

### 4.2 免费用户限制

| 限制 | 说明 |
|------|------|
| **辩论记录** | 最多保存5条，超出自动删除最早的 |
| **群组** | 最多创建3个 |
| **Soul好友** | 无限制 |
