## 三、色彩系统

**核心规则**：严格按照微信的UI界面设计，微信使用绿色的地方，才使用#AFDD22颜色，其他地方配色和组件，要完全与微信一致。

| 微信原色用途 | 微信原色 | PRD辩论APP新色 | 说明 |
|--------------|----------|---------------|------|
| 导航栏背景 | #07C160 | `#AFDD22` | 顶部导航栏背景 |
| 底部Tab选中 | #07C160 | `#AFDD22` | 选中的Tab图标和文字 |
| 按钮/链接 | #07C160 | `#AFDD22` | 可点击元素的高亮色 |
| 成功提示 | #07C160 | `#AFDD22` | 操作成功提示 |
| **其他场景** | **#07C160** | **保持原色或灰色** | 不需要强调的地方 |

| 用途 | 颜色 | Hex |
|------|------|-----|
| **主色调** | 黄绿色 | `#AFDD22` |
| 消息气泡（用户） | 浅绿 | `#95EC69` |
| 消息气泡（AI） | 白色 | `#FFFFFF` |
| 背景色 | 浅灰白 | `#F5F5F5` |
| 文字主色 | 深灰黑 | `#1F1F1F` |

## 五、消息设计

### 消息时间戳
- **不显示时间**

### 消息状态
- 模型连通时：显示"**正在输入中...**"
- 模型断开时：显示"**用户离线**"
- 其他情况：不显示

### AI角色头像
- **引入外部资源**
- 下载网上的头像资源
- 不能使用emoji或固定图标

### 消息气泡样式
- AI消息（左侧）：白色背景 #FFFFFF，圆角8px
- 用户消息（右侧）：浅绿背景 #95EC69，圆角8px

## 十、加载与错误处理

### 首次加载状态
- **加载动画**（从外部引入和下载动画资源）

### 空状态设计
- **显示引导文字+按钮**

| 场景 | 处理方式 |
|------|----------|
| 会话列表为空 | 显示"暂无讨论记录" + **发起讨论**按钮 |
| 通讯录为空 | 显示"暂无群组或好友" + **创建群组/添加好友**按钮 |

### 错误提示
- **Toast提示 + 页面内提示**

| 场景 | 处理方式 |
|------|----------|
| 一般错误 | Toast提示（底部弹出，自动消失） |
| 严重错误 | 页面内提示（显示错误详情和操作按钮） |

## 十一、消息字数控制

### 核心要求

| 要求 | 说明 |
|------|------|
| **推荐范围** | 100-300字/条消息 |
| **表达风格** | 简洁自然，像朋友聊天 |
| **语气词** | 可用"嗯""啊""好吧""其实"等口语 |
| **核心观点** | 提炼关键信息，不废话 |

### 禁止的表达方式

| X AI机械表达 | 人类自然表达 |
|--------------|----------------|
| "作为一个AI，我认为..." | "我觉得吧..." |
| "从多个角度来看..." | "其实吧..." |
| "第一，第二，第三..." | "首先然后..." |
| "综上所述..." | "反正就是这么回事" |
