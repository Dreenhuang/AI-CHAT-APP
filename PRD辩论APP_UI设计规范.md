## 一、设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **一致性** | 所有界面遵循微信UI的设计语言 |
| **复刻度** | 100%复刻微信的尺寸、文字大小、间距、交互 |
| **主题色** | 仅在微信使用绿色的地方，替换为#AFDD22 |
| **对话风格** | 松弛自然，像朋友聊天，拒绝AI机械表达 |

### 1.2 微信Tab导航映射

| 微信Tab | 微信内容 | PRD辩论APP映射 |
|--------|----------|----------------|
| **微信** | 会话列表 | 历史讨论记录列表 |
| **通讯录** | 好友 + 群聊 | 辩论群组 + Soul好友（群组在前，可拖拽排序） |
| **发现** | 朋友圈/小程序 | AI预设200议题，随机推送20个，下拉筛选分类 |
| **我** | 个人中心 | 管理/设置页面 |

## 二、色彩系统

### 2.1 主题色替换规则

**核心规则**：严格按照微信的UI界面设计，微信使用绿色的地方，才使用#AFDD22颜色，其他地方配色和组件，要完全与微信一致。

| 微信原色用途 | 微信原色 | PRD辩论APP新色 | 说明 |
|--------------|----------|---------------|------|
| 导航栏背景 | #07C160 | `#AFDD22` | 顶部导航栏背景 |
| 底部Tab选中 | #07C160 | `#AFDD22` | 选中的Tab图标和文字 |
| 按钮/链接 | #07C160 | `#AFDD22` | 可点击元素的高亮色 |
| 成功提示 | #07C160 | `#AFDD22` | 操作成功提示 |
| **其他场景** | **#07C160** | **保持原色或灰色** | 不需要强调的地方 |

### 2.2 完整色彩系统

| 用途 | 颜色 | Hex | 说明 |
|------|------|-----|------|
| **主色调** | 黄绿色 | `#AFDD22` | 仅用于微信绿色的地方 |

| Tab选中 | 黄绿色 | `#AFDD22` | 底部Tab选中状态 |

## 五、页面结构

### 5.1 底部Tab导航（4个Tab）

**Tab图标风格**：纯文字（类似微信）

```
┌─────────────────────────────────┐
│  [+]      [++]      [💡]      [++]  │
│  微信     通讯录    发现       我      │
│ (#AFDD22)  (灰色)  (灰色)     (灰色)  │
└─────────────────────────────────┘
```

**说明**：
- Tab图标使用纯文字，不使用emoji
- "发现"Tab使用灯泡图标，需要引入外部资源（不能使用emoji）
- 选中状态：图标和文字颜色变为#AFDD22

### 5.2 页面清单

| Tab | 页面名称 | 功能 |
|-----|----------|------|
| Tab1-微信 | 会话列表 | 历史讨论记录列表 |
| Tab2-通讯录 | 通讯录 | 群组列表 + 好友列表（可拖拽排序）|
| Tab3-发现 | 发现 | AI预设议题，下拉筛选分类 |
| Tab4-我 | 我的 | 个人中心、管理设置 |

## 八、字数控制规范

### 8.1 核心要求

| 要求 | 说明 |
|------|------|
| **推荐范围** | 100-300字/条消息 |
| **表达风格** | 简洁自然，像朋友聊天 |
| **语气词** | 可用"嗯""啊""好吧""其实"等口语 |
| **格式化** | 不需要规整排版，不用过度严谨 |
| **核心观点** | 提炼关键信息，不废话 |

### 8.2 禁止的表达方式

| X AI机械表达 | 人类自然表达 |
|--------------|----------------|
| "作为一个AI，我认为..." | "我觉得吧..." |
| "从多个角度来看..." | "其实吧..." |
| "第一，第二，第三..." | "首先然后..." |
| "综上所述..." | "反正就是这么回事" |
| "需要指出的是..." | "你要知道..." |

## 九、加载状态与错误处理

### 9.1 首次加载状态

**选择：B. 加载动画**

| 场景 | 处理方式 |
|------|----------|
| **首次加载** | 显示加载动画（从外部引入和下载动画资源） |
| 网络请求中 | 显示加载动画 |
| 页面切换 | 显示加载动画 |

**动画资源**：
- 需要引入外部动画资源
- 从网上下载动画文件（如Lottie格式）
- 不能使用简单的loading文字

### 9.2 空状态设计

**选择：A. 显示引导文字+按钮**

| 场景 | 处理方式 |
|------|----------|
| 会话列表为空 | 显示"暂无讨论记录" + **发起讨论**按钮 |
| 通讯录为空 | 显示"暂无群组或好友" + **创建群组/添加好友**按钮 |
| 发现页为空 | 显示"暂无议题" + **刷新**按钮 |

### 9.3 错误提示

**选择：A. Toast提示 + C. 页面内提示**

| 场景 | 处理方式 |
|------|----------|
| 一般错误 | Toast提示（底部弹出，自动消失） |
| 严重错误 | 页面内提示（显示错误详情和操作按钮） |
| 网络错误 | Toast提示"网络连接失败" |
| 业务错误 | 页面内提示（如"群组创建失败"） |
