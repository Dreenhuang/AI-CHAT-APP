## 一、深入理解与分析

### 1.1 当前系统定位

当前系统的逻辑是 **"Proposer → Reviewer → 下一轮"** 的线性模式，本质上是：

```
提案者输出方案 → 审查者评审 → 固定轮次 → 进入下阶段
```

这种模式非常适合 **PRD文档评审**，因为：
- 提案者 = 文档作者
- 审查者 = 评审专家
- 线性流程 = 文档评审流程
- 固定轮次 = 评审轮次

### 1.2 讨论形式分类理解

根据文档，六大类讨论形式的核心特征：

| 类型 | 核心特征 | 典型场景 |
|------|---------|---------|
| 一对一双向商量 | 平等、简单、无规则 | 两人讨论 |
| 多人圆桌合议 | 秩序化、全面覆盖 | 团队研讨 |
| 正式对抗辩论 | 有立场、有攻防 | 方案优劣辩论 |
| 结构化议事决策 | 高效落地、可执行 | 决策投票 |
| 头脑风暴共创 | 发散、创意、无限 | 创意收集 |
| 多AI协同 | 专业分工、全面无盲区 | AI协同讨论 |

### 1.3 每种讨论模式的逻辑结构

#### 类型一：一对一双向商量（2方）

**1. 双向自由对谈式**
```
顺序：A完整输出 → B完整输出 → 双方商讨 → 结论
逻辑：各说各的，最后讨论收敛
```

**2. 一问一答追问式**
```
顺序：A提问 → B回答 → A追问 → B再答 → ...
逻辑：问答链式，直到信息差消除
```

**3. 互补完善式**
```
顺序：A输出 → B只补漏洞 → A整合 → B再审 → 共同确认
逻辑：B不重复A内容，只补盲区
```

#### 类型二：多人圆桌合议（3方+）

**4. 圆桌自由研讨式**
```
顺序：无固定顺序，自由发言，主持人协调
逻辑：发散式，各方自由表达观点
```

**5. 轮值发言合议式**
```
顺序：每人轮到发言，其他人聆听，循环直到共识
逻辑：秩序性确保每个人都表达
```

**6. 分头立论再汇总式**
```
顺序：主持人分维度 → 各方独立输出 → 互相审阅 → 补充 → 主持人汇总
逻辑：先分工输出，再整合
```

**7. 分工专项研讨式**
```
顺序：立论方 → 补漏方 → 挑错方 → 立论方修正 → 总结方 → 最终确认
逻辑：角色分工明确，流程严谨
```

#### 类型三：正式对抗辩论（3方+）

**8. 标准正反方辩论赛制**
```
顺序：正方立论 → 反方立论 → 自由辩论 → 总结
逻辑：有立场对立，正反对抗
```

**9. 质询答辩辩论制**
```
顺序：立论方 → 质询方提问 → 立论方答辩 → 多轮质询
逻辑：质询方挑漏洞，立论方辩护
```

**10. 三方三角辩论制**
```
顺序：正方 → 反方 → 中立方 → 三角对抗 → 中立汇总
逻辑：正反+裁判结构
```

**11. 驳论复盘辩论制**
```
顺序：立论方 → 多方驳论 → 立论方修正 → 无漏洞后总结
逻辑：多方挑战，逼出漏洞
```

#### 类型四：结构化议事决策（3方+）

**12. 提案表决式**
```
顺序：主持人提出方案 → 各方表决 → 少数服从多数
逻辑：高效决策，民主投票
```

**13. 问题拆解逐级研讨式**
```
顺序：大问题 → 拆解子问题 → 逐个讨论 → 汇总
逻辑：复杂问题分解为简单问题
```

**14. 优缺点分列合议式**
```
顺序：罗列优点 → 罗列缺点 → 权衡分析 → 结论
逻辑：黑白分明，客观评估
```

#### 类型五：头脑风暴共创（2方+）

**15. 发散头脑风暴式**
```
顺序：所有人自由发言 → 收集所有创意 → 无批评 → 最后收敛
逻辑：发散优先，收敛在后
```

**16. 创意接龙讨论式**
```
顺序：一人起头 → 下一人接力 → ... → 迭代优化
逻辑：创意接力，层层叠加
```

#### 类型六：多AI专属协同（3方+）

**17. 主AI牵头+副AI补位式**
```
顺序：主AI主导 → 副AI补充 → 主AI汇总
逻辑：有一个核心大脑
```

**18. 平行AI独立输出再整合式**
```
顺序：各AI独立输出 → 汇总 → 整合
逻辑：无干扰，独立思考后合并
```

**19. 角色模拟合议式**
```
顺序：主AI分配角色 → 各角色AI独立发表专业意见 → 互相提问 → 主AI汇总
逻辑：专业角色分工，全面覆盖
```

## 二、现有系统分析

### 2.1 当前实现的模式

当前系统近似于 **"分工专项研讨式"** 的变体：
- 主持人 = 协调者
- 提案者 = 立论方
- 审查者 = 补漏方+挑错方

### 2.2 需要改造的点

| 组件 | 当前状态 | 需要改造 |
|------|---------|---------|
| 讨论引擎 | 固定Proposer→Reviewer循环 | 需要支持多种讨论逻辑 |
| 角色管理 | 3个固定角色 | 需要动态角色池（30+角色） |
| 前端UI | 固定模式选择 | 需要模式选择器 |
| 输出控制 | 无字数控制 | 需要三种输出深度 |
| 显示样式 | 线性消息流 | 需要根据模式定制 |

### 阶段三：Soul角色设计（30+角色）

#### 分类设计

**主持人类（5个）**：
1. 中立协调型 - 公平引导，不偏袒
2. 效率优先型 - 追求快速结论
3. 温和鼓励型 - 鼓励所有人发言
4. 严格控场型 - 确保按规则进行
5. 创意激发型 - 引导发散思维

**提案方/立论方（5个）**：
1. 逻辑严谨型 - 数据支撑，论证严密
2. 创新激进型 - 提出突破性想法
3. 务实可行型 - 注重落地性
4. 理论深度型 - 学术性强
5. 经验丰富型 - 实战经验驱动

**审查/质疑方（5个）**：
1. 挑剔细节型 - 关注每个漏洞
2. 风险意识型 - 重点关注风险
3. 成本效益型 - 关注投入产出
4. 用户导向型 - 关注用户体验
5. 技术可行型 - 关注技术实现

**补位/补充方（5个）**：
1. 全面覆盖型 - 补充所有盲区
2. 细节丰富型 - 关注细节完善
3. 案例驱动型 - 用案例补充
4. 扩展思考型 - 延伸更多角度
5. 实操落地型 - 注重可执行性

**总结方（5个）**：
1. 简明扼要型 - 简洁明了
2. 系统整合型 - 系统性总结
3. 行动导向型 - 给出可执行建议
4. 风险预警型 - 强调风险点
5. 共识构建型 - 确保达成共识

**辩手/对抗方（5个）**：
1. 进攻型 - 主动挑战
2. 防守型 - 善于防守反驳
3. 逻辑型 - 逻辑攻击
4. 情感型 - 情感诉求
5. 综合型 - 攻防兼备

**头脑风暴角色（5个）**：
1. 点子王型 - 创意无限
2. 批判型 - 防止过度乐观
3. 执行型 - 关注实现
4. 用户型 - 代表用户声音
5. 商业型 - 关注商业价值
